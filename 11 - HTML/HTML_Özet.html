1. DERS

Tag Nedir?

HTML komutuna tag denir. Tag'lar büyüktür kücüktür isareti arasina yazilir. Istisnalar disinda tag'larin baslangic ve bitisleri vardir. Bitis taglari / isareti ile belirtilir.
Istisna: img, br ve hr tag'larinin bitis isareti yoktur.

Nasil HTML Sayfasi Olusturabiliriz?

.html uzantili bir dosya olusturarak html sayfasi olusturmaya baslayabiliriz.

<html>

html uzantili dosyasi actiktan sonra basa ve sona html tag'ini yazarak baslamamiz gerekir. Bundan sonra gelecek olan tüm komutlar bu iki html tag'i arasina gelir.

<head>

Sayfa ile ilgili ön bilgiler, ayarlar head tag'inda bulunur. head tag'inda olmasi gereken en önemli tag title tag'idir.

<title>

Head tag'i icinde bulunur ve tarayicinin baslik cubugunda görüntülenecek metni belirler. SEO acisindan cok degerlidir.

<body>

Kullanicinin görmesini istedigimiz her seyi body tag'ina yazariz.

Element

Tag ve icindekilerle birlikte olusan yapiya element denir. Örnegin title tag'i ile birlikte yazdigimiz HTML Özet ifadesi bir element olusturur.

HTML5

HTML kodunun HTML5 oldugunu en bastaki !DOCTYPE html tag'indan anlariz.

White Spaces

Tarayicilar, HTML kodlarindaki pes pese space (bosluk) karakterlerini tek bir karakter olarak yorumlarlar. Ayrica enter (alt satira gecme) karakterlerini ise hic yorumlamazlar. Bu sayede developerlar kodlarini daha okunabilir sekilde yazabilirler.

2. DERS

Semantic ve Non-Semantic

Semantic tag'lar, arama motorlari veya gelistiriciler icin bir anlam icerirler. Bu tür tag'lar kapsadiklari elemani tasidiklari anlama göre belli bir forma sokarlar. Paragraf tag'i bunlara örnek verilebilir.

Semantic olmayan tag'lar, anlamsizdirlar. Sadece semantic tag'lari gruplamak icin kullanilirlar. Arama motorlari icin de bu tag'larin hicbir degeri yoktur. <div> ve <span> tag'lari semantic olmayan tag'lara örnek verilebilir.

<strong> ve <b>

Ikisinin de görevi metni kalinlastirmaktir.

Aralarindaki fark, strong tag'i b tag'ina göre daha semantic'tir yani arama motorlari tarafindan sevilir. Örnegin arama motorlarinin dikkatini cekmek icin strong tag'ini, kullanicinin dikkatini cekmek icin b tag'ini kullanabiliriz.

<em> ve <i>

Ikisi de yaziyi egik yazdirmak icin kullanilirlar.

em tag'i semantic olarak daha degerlidir.

<u>

Yazinin altini cizmek icin kullanilir.

Sayfaya Ses Ekleme

<audio> tag'i ile sayfaya ses ekleyebiliriz. Ses dosyalari mp3, wav, ogg gibi formatlarda olabilir. Ama tarayicilarin en cok destekledigi format mp3'tür. Telifsiz müzikleri indirmek icin pixabay sitesi kullanilabilir. Ses dosyalarinin kullanici deneyimi acisindan olumsuzluklar olusturmasi nedeniyle modern tarayicilar ses dosyalarinin direkt siteye girer girmez baslamasini istememektedir o yüzden kod icerisinde controls parametresi eklenmelidir.

Sayfaya Video Ekleme

<video> tag'i ile sayfaya video ekleyebiliriz. Video dosyalari MP4, WEBM, OGG gibi formatlarda olabilir. Telifsiz videolari indirmek icin pexels sitesi kullanilabilir. Video dosyalarinin kullanici deneyimi acisindan olumsuzluklar olusturmasi nedeniyle modern tarayicilar video dosyalarinin direkt siteye girer girmez baslamasini istememektedir o yüzden kod icerisinde controls parametresi eklenmelidir.

<iframe>

Bir web sayfasinin icine baska bir web sayfasi gömmeye yarar. frameborder parametresi cerceveyi belirler ve sadece 0 (cerceveyi gizler) ve 1 (cerceveyi gösterir) degerlerini alabilir. HTML5 ile birlikte kullanilan bir parametre degildir. Modern yaklasim cerceve kontrolü icin CSS kullanimini tesvit etmektedir.

Tablo Olusturma

Tablo olustururken 7 tag kullanilabilir:
  • <table> => Tablo olusturmak icin kullanilir.
  • <thead> => Tablo baslik satirlarini gruplayan bir alan olusturur.
  • <tbody> => Tablo icerigini (veri satirlarini) gruplamak icin kullanilir.
  • <tfoot> => Tablo alt kismini (genellikle toplamlar veya ek aciklamalar gibi) olusturmak icin kullanilir.
  • <tr> => Tablo satiri olusturmak icin kullanilir.
  • <td> => Veri hücresi olusturmak icin kullanilir
  • <th> => Baslik hücreleri olusturmak icin kullanilir.

NOT: thead, tbody, tfoot kullanmak zorunlu degildir ancak CSS ile stil tanimlarken büyük kolaylik olur.

NOT: table icerisine yazdigimiz border kirmizi gözüküyor cünkü eski bir özellik, artik cerceveyi HTML ile degil CSS ile yönetiyoruz. border parametresi 0 (cerceve yok) veya 1 (cerceve var) degerini alabilir.

Form Olusturma

  • <form> => Tüm form elemanlarini icinde barindiran tasiyicidir.
  • <input> => Kullanicidan bilgi almak icin kullanilan form elemanlarinin genel ismidir.
  • <label> => Kullaniciya form elemani ile ilgili bilgi vermek icin kullanilan elemandir.

Form tag'inin sahip oldugu cesitli attribute'lar vardir:
  • action => Form bilgileri ile sunucudaki hangi endpoint ilgilenecekse o belirlenir.
  • method => Genellikle post veya get degeri alir. Formdaki bilgilerin sunucuya hangi yöntemle gönderilecegini belirler.
  • enctype => Gönderilecek datanin nasil sifrelenecegi (endcode) belirlenir.
  • novalidate => HTML5 gelen yerlesik dogrulamayi devre disi birakir.

input'un type'ina farkli girisler yazabiliriz:
  • text => Kullanicidan herhangi bir metinsel bilgi almak icin kullanilir.
  • number => Kullanicidan sayisal bilgi almak icin kullanilir. Metin girisine izin vermez.
  • e-mail => e-mail girisi gerceklestirilmesini saglar. Görünüste text'ten farki yoktur.
  • password => text'ten farki girileni gizlemesidir.
  • search => Arama yapmak icin kullanilir. text'ten farki sag tarafta carpi cikmasidir.
  • url => Görünüs olarak text'ten farki yoktur, URL formunda giris olmazsa uyari verir.
  • tel => Görünüs olarak text'ten farki yoktur, telefon formatinda giris olmazsa uyari verir.
  • hidden => Kullanicinin görmesine gerek olmayan alanlari hidden yapariz.

3 tip buton vardir:
  • button => Özellestirilebilir butondur.
  • submit => Özelligi belli buton türüdür: action'deki yola gönderimi saglar.
  • reset => Özelligi belli buton türüdür: Formu ilk haline cevirir.

NOT: Form icindeki her bir input ögesine bir name parametresi verilir, bu parametre sunucuya gönderilen verilerde anahtar olarak kullanilir. Kullanicinin girdigi degerler value olarak kabul edilir ve key value eslesmesi seklinde sunucuya gönderilir.
